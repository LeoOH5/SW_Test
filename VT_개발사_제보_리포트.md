## 1. 🔴 심각 (Critical)

### V-CRIT-1 · 미게시(active=false) 스페이스 콘텐츠가 비인증 뷰어 API로 유출
- **재현**: `active=false` 스페이스에 씬 생성 후 **토큰 없이** `GET https://vtapi.visitinc.kr/vt?VT=<7자리hex>` → **HTTP 200**으로 스페이스 제목·씬 정보·**360 타일 저장 경로**·**원본 업로드 파일명** 반환.
- **추가 확인**: 미게시 스페이스의 **defaultThumb(썸네일)도 비인증 200**으로 노출.
- **정밀화(정직성)**: 타일 **본체** URL(`business.roundpicapp.com/storage/VT/...`, 서명 URL)은 비인증 **403**으로 보호됨 → 타일 이미지 대량 다운로드는 불가. 그러나 **미게시 콘텐츠의 존재·구성·경로·썸네일·파일명 메타데이터가 공개**됨.
- **영향**: 게시 여부와 무관하게 VT 키만 알면 미공개 스페이스 열람. 키는 7자리 hex라 열거 리스크 존재(열거 가능 범위는 미검증 §6).
- **수정 제안**: `/vt` 응답을 **게시상태(active)로 게이팅**. 썸네일도 동일 게이팅. VT 키 엔트로피 강화.
- 증적: `recon/verify_all_fresh.md`, `recon/verify_v1`, `recon/full_audit.json`(A01X).

### V-CRIT-2 · 배치 업로드에 미지원 포맷 1장 혼입 시 정상 파일까지 전량 유실 (데이터 손실)
- **재현(수동 통제실험 8종, 재현율 100%)**: 다중선택 배치에 **거부 포맷(WebP) 1장**만 섞여도 같은 배치의 **정상 파노라마까지 전부 등록 실패**. 실패 사유 안내 없이 "업로드 실패" 토스트 + 빈 유령 카테고리 잔존.
- **근본원인(자동 API 교차검증으로 로커스 특정)**: 직접 API 호출에서는 **정상 씬이 서버에 정상 등록·유지**됨(webp 거부와 무관). 즉 정상분 유실은 **서버 롤백이 아니라 프론트엔드 배치 완료 핸들러 버그** — 거부 파일이 반환하는 **빈 jobId**를 집계하다 깨지며 성공분 씬 확정을 못 하는 것으로 추정.
- **트리거 조건**: WebP는 게이트에서 **빈 jobId(code203)** 반환이 트리거. (GIF는 게이트 통과 후 비동기 거부라 트리거 여부 별도 확인 권장.)
- **영향**: 사진 폴더 통째 드래그 시 스크린샷 WebP 1장만 섞여도 정상 파노라마 전부 유실.
- **수정 제안**: 배치 완료 핸들러가 개별 파일 실패를 **격리 처리**(부분 성공 허용) + 실패 사유·대상 파일 고지. 클라이언트 업로드 큐에서 빈 jobId 방어.
- 증적: 통제 매트릭스(v5 §E-018), `recon/verify_v5.json`.

## 2. 🟠 보통 (Major)

### V-MAJ-1 · 로그인 사용자 열거 + 실패에 성공 코드(HTTP 201)
- 없는 계정 → `201 {"code":101,"message":"id not exist"}` / 가입계정+틀린비번 → `201 {"code":102,"message":"pwd not compare"}`. **응답 코드로 계정 존재 여부 구분** = 사용자 열거. 인증 실패를 2xx로 통보.
- 수정: 실패 시 **401 + 단일 일반 메시지**(계정존재/비번오류 구분 제거).

### V-MAJ-2 · 입력검증 미비로 4xx 대신 HTTP 500
- 초장문 파일명(UTF-8 ~255B 초과, 한글 100자↑) 업로드 → `POST /scene/autoNorth` **500**. `PUT /content`에 존재X/불완전 body → **500**(생성 `POST /content`는 400 검증 존재 → **검증 비대칭**). 단, 500 응답 본문은 스택 미노출(양호).
- 수정: 전 엔드포인트 일관된 입력검증(400)·예외 핸들링.

### V-MAJ-3 · 보안 응답 헤더 6종 전무 + 토큰 localStorage 평문
- 웹·API 모두 **CSP·X-Frame-Options·HSTS·X-Content-Type-Options·Referrer-Policy·Permissions-Policy 부재**. `x-powered-by: Express` 노출. **accessToken·refreshToken이 localStorage 평문** 저장 → CSP/헤더 부재와 결합 시 XSS로 세션 탈취 표면 확대.
- 수정: 표준 보안 헤더 세트 적용, `x-powered-by` 제거, 토큰은 httpOnly+Secure 쿠키 검토.

### V-MAJ-4 · 특정 해상도 파노라마 업로드 처리 실패 (콘텐츠 무관)
- 정상 규격(2:1·유효 JPEG)인데 **≈10771×5385 해상도는 처리 실패**(jobId 발급 후 씬 미등록), 7180×3590·11968×5984는 통과. **해상도만 스왑하면 성패 반전**(A/B 통제 + 자동 재현 확정). 실패 사유·허용 해상도 안내 없음.
- 수정: 처리 가능 해상도 상한 명시·사전검증·실패 사유 고지.

### V-MAJ-5 · 초소형(100×50) 이미지가 씬으로 통과 → 뷰어 검은 화면
- 최소 크기/비율 검증 부재. API에서 tiny 씬 생성 + 뷰어 완전 검은 화면(양쪽 확인). 안내·플레이스홀더 없음.

### V-MAJ-6 · 업로드 용량 상한이 안내와 5배+ 불일치
- 안내 "5MB까지"인데 실측 **24MB까지 통과**, 35MB 거부. 상한 미정의·안내 오류.

### V-MAJ-7 · 업로드 직후 에디터가 "이미지 없음"으로 오표시(새로고침 필요)
- 비동기 job 반영 지연. 육안 + 자동(렌더 캡처 차단) 양쪽 확인.

### V-MAJ-8 · HTTP 상태코드 시맨틱 불일치
- 업로드 거부(code203)를 **202**로, 인가 실패(code103)를 **200**으로 반환 → 클라이언트/모니터링 오판 유발.

## 5. ✅ 테스트했고 정상(PASS) — 커버리지 증빙 (오탐 방지)
개발사 신뢰를 위해, 점검했으나 문제없던 항목을 명시한다.
- **권한/IDOR(수평)**: `/company`·`/content/list` 미소속 회사 → `code103` 차단. `/company/list`는 본인 2개 회사만. `/user?id=`는 파라미터 무시하고 본인 반환(타인 조회 아님).
- **인증**: 비인증 API → 401. 로그아웃 → 서버측 토큰 무효화. 토큰은 opaque(JWT 아님, alg 변조 불가).
- **주입**: SQL/NoSQL 페이로드(`' OR 1=1`, `{$ne:null}`) 방어(파라미터 검증). 타입 혼동(배열/객체 주입) → 400.
- **업로드**: 나쁜 파일(0바이트·손상·텍스트위장·webp·bmp) 최종 거부. SVG 거부. path traversal 파일명은 basename만 사용. **타일 본체 스토리지 비인증 403**.
- **정보노출**: 소스맵·.env → 404. 500 응답 스택 미노출.
- **검증**: POST /content 생성 검증(400), 좌표/수치 검증(400), 회사명 길이 상한(400).

## 6. ⏸ 아직 검증 못 한 고위험 항목 (개발사 협의/제2계정 필요)
아래는 **계정 잠금·타 고객 데이터·서비스 부하·게시상태 변경 위험** 때문에 인가된 안전범위에서 **의도적으로 실행하지 않음**. 개발사 동의·제2계정 확보 후 통제된 환경에서 추가 검증 권장.
1. **계정 잠금 / 로그인 rate limit 부재 여부**(V-MAJ-1 열거와 결합 시 브루트포스) — 테스트 계정 잠금 위험.
2. **객체단위 IDOR**(scnId/hsId/cntId/rsviId/ucmpnrId 타인 객체 조회·수정·삭제) — 제2계정 필요, 쓰기 IDOR는 데이터 파괴 위험.
3. **VT 키 열거 범위**(7자리 hex로 타 고객 미게시 스페이스 접근 가능성) — 실 타고객 데이터라 법적·윤리 위험, 개발사 입회 필요.
4. **Mass assignment**(`PUT /content`/`/company`에 `active:true`·`cmpnId`·`role` 주입으로 게시상태·소유권·권한 강제) — **active=false 유지 위반 위험**으로 미수행. 성공 시 V-CRIT-1 게이팅·회사제한·격리 우회 → 최우선 협의 대상.
5. **디코딩 DoS**(픽셀폭탄 초고해상도)·**회사 10개 제한 우회**·**active 토글 인가**·**경쟁상태 중복생성**.
6. **저장형 XSS 뷰어 실행**(V-MIN-N1) — `PUT /content`가 유효 cntId에도 500이라 API로 뷰어필드 주입 불가 → 에디터 UI로 사람이 실행 검증 필요.

## 7. 방법·정직성
