## 2. 3관점 분업 — 겹치지 않게 커버리지 구멍 없애기

| 관점 | 핵심 질문 | 노리는 배점 | 대표 결함유형 |
|---|---|---|---|
| **QA** | "명세대로·빠짐없이·산출물이 규칙대로인가?" | 커버리지10+TC10+보고서10+공식기법10 = **40** | 명세 불일치·상태 정합성·표기 오류 |
| **TE** | "기술적으로 부술 경계·보안·성능 취약점은?" | **결함 50** + 창의기법10 | Breakdown 크래시·검증부재·IDOR·콘솔에러 |
| **User** | "실사용자가 겪는 불편·혼란·불신은?" | 결함(경미·개선) + 보고서 설득력 | 안내부재·피드백없음·프라이버시노출·불필요경고 |

> TE가 심각·보통(50점 핵심) 사냥, QA가 빠짐없음+명세일치+산출물을 굳힘, User가 설득력 있는 경미·개선으로 폭·완성도 채움. **서로 다른 근본원인**으로 결함 총합을 saturate.

## 3. 오늘(D3, 7/15) 실행 계획 — 관점별 작업

> 남은 미완(P1~P4)이 업로드 실패로 막혔던 상태 → **재로그인으로 업로드 신뢰성 먼저 회복**한 뒤 TE 사냥 재개. 병행으로 QA 커버리지·User 시나리오 진행.

### 3-A. TE 관점 (최우선 — 심각·보통 50점)
1. **재로그인 → 업로드 신뢰성 확인**: 정상 08MB 2:1 파일 단일/다중 업로드 → 반복 실패면 "업로드 신뢰성" 결함, 정상 복구면 아웃티지는 환경으로 분리(섣부른 결함화 금지).
2. **P3 ★ E-008 크래시 격리**: 빈 스페이스에 변형이미지 1장씩 업로드 → white screen 유발 파일/조합 특정 → 재현절차 + 콘솔(`null children`)·스크린샷 증적. **심각 후보 방어의 핵심.**
3. **P1 E-010 용량 상한 이진탐색**: `X3\용량경계\` 06~24MB 작은 것부터 → 통과↔거부 전환점 = 실제 상한. 안내(5MB) vs 실제(24MB+) 불일치 강화.
4. **P2 E-009 비율·해상도 검증부재**: 정사각/16:9/세로 각 단독 업로드 → 왜곡/검은화면 확정.
5. **포맷 개별**: png(통과 확인)/webp/gif/bmp 단독 업로드 통과·차단 확정.
6. **권한**: IDOR(관리 URL `cntId` 타 스페이스 조작), 로그아웃 후 뒤로가기 관리화면 접근, 세션 만료 후 저장/업로드.
7. **콘솔·성능**: E-015 상시 콘솔에러(F12) 증적 / E-018 카드 오진입 해결 후 P4 37씬 성능(로딩·씬이동·프리즈).

### 3-B. QA 관점 (안전 40점 굳히기 — 병행)
1. **커버리지 빈 소분류 PASS 메우기**: 비밀번호찾기 / 내정보(이름·전화 변경) / 플랜조회 / 지도좌표 / 공유·링크 / 뷰어 씬목록·지도토글. PASS 19 → 30+ 목표.
2. **TC 완성도**: 기대결과 구체화("정상 처리"→"메인 이동+상단 Login→로그아웃"), 전제조건·적용 기법 태그, D1 골격 행의 실제결과/판정 채움.
3. **명세 정합성 결함 재현·증적**: E-002 토스트 띄어쓰기 / E-003 도움말 4종 vs 메뉴 5종 / E-005 카테고리 명명 불일치 / E-012 업로드 직후 빈 상태 / URL cntId 라우팅 정합성.

### 3-C. User 관점 (경미·개선 + 설득력 — 병행)
1. **프라이버시**: E-014 원본 파일명 노출(뷰어 헤더 `X3_H-1(1).jpg`) — active OFF 상태에서 관찰, 공개뷰어 확정은 개발사 확인 후.
2. **피드백 부재**: E-006 공유 버튼 무반응 / E-013 예약 핫스팟 미선택 저장 무반응 → "작업 결과 피드백 없음" 묶음.
3. **혼란 유발 UX**: E-001 변경없이 취소해도 삭제경고 / E-007 모달 재오픈 시 이전 입력값 잔존 / E-004 용량·포맷 안내 부재.
4. **실사용 워크스루**: 회사→스페이스→업로드→씬→핫스팟→(발행 전)확인 1회 관통, 이탈지점·안내부족 기록.
