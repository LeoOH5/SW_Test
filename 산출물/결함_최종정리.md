# VT partners 결함 최종 정리

> 재검증일 2026-07-19 · 계정 `dhwjdqls0409@korea.ac.kr` · 회사 cmpnId=100
> 직접 실측(curl API 호출 + 실제 파일 업로드 + 실브라우저 콘솔 관찰) 기준으로 3분류.

---

## ① 진짜 존재하는 결함 (직접 확인함, 반박 어려움)

1. **긴 파일명 업로드하면 서버가 500 에러로 터짐**
   짧은 이름은 정상, `가`×120 이름만 넣어도 서버가 뻗음. 정상이면 400으로 막아야 함.
2. **로그인 실패 메시지로 가입된 이메일인지 알아낼 수 있음**
   없는 계정은 "id not exist", 있는 계정+틀린 비번은 "pwd not compare"로 응답이 달라 공격자가 가입 이메일을 캐낼 수 있음. 게다가 실패인데 성공 코드(201)를 줌.
3. **기본 보안 헤더가 하나도 없음 + 평문(http) 접속이 그대로 열림**
   암호화 안 된 http 주소로 들어가도 차단·리다이렉트 없이 열림.
4. **업로드가 파일 내용을 검사 안 함**
   텍스트 쓰레기 파일을 "이거 이미지야"(MIME만 image/jpeg)라고 우기면 서버가 그대로 받아줌.
5. **없는 주소로 들어가면 404 안내 없이 에러(NG04002)만 뜸**
   브라우저에서 직접 확인. 사용자용 안내 페이지가 없음.
6. **홍보링크 칸에 URL이 아닌 아무 글자나 저장됨**
   `not_a_valid_url_링크` 같은 값이 검증 없이 그대로 들어감.
7. **welcomeMessage에 문자열 "undefined"가 저장됨**
   스페이스 3개 전부. 프론트 버그로 보임.

---

## ② 결함인지 사양인지 확인 필요 (개발사에 물어봐야 함)

1. **허용 이미지 포맷이 이상함**
   gif·png는 받고 webp·bmp는 거부. 의도한 사양인지(파노라마에 gif 애니메이션 허용?) 확인 필요.
2. **파노라마 비율(2:1)을 안 지켜도 업로드가 통과됨**
   게이트는 정사각·16:9·세로도 받음. 단 최종적으로 씬은 안 만들어짐. 일부러 느슨하게 둔 건지 확인 필요.
3. **welcomeMessage "undefined"가 어디서 생기는지**
   저장된 값은 확실하나, 어떤 입력 경로에서 생기는지는 개발사 코드 확인 필요.

---

## ③ 내가 검증 못 하는 것 (사람이 눈·손으로 해야 함)

1. **360 화면이 실제로 찌그러지는지** — 비율 안 맞는 실사 사진이 뷰어에서 왜곡돼 보이는지 사람 눈 확인.
2. **보안헤더 없는 게 실제 클릭재킹으로 뚫리는지** — iframe에 끼워 공격 성공하는지 실증 필요.
3. **핫스팟 꼬임** — 이동 핫스팟 걸어둔 씬을 삭제하면 링크가 깨지는지.
4. **삭제 잔존** — VT/스페이스 삭제 후 목록·링크에 남는지.
5. **저장형 XSS** — 제목·설명에 악성 스크립트 넣으면 실행되는지 (실서비스 데이터라 신중히).
6. **로그아웃 후 뒤로가기·세션만료** 시 관리화면 접근되는지.
7. **비밀번호 찾기 이메일** 실제 발송·문구 (사람이 1회만).
8. **브라우저 호환·모바일·느린 네트워크**에서 대용량 파노라마 로딩.
