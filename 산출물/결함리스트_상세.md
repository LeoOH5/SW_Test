## 심각 (Critical)

### E-001 — 초소형(100×50) 파노라마 업로드 시 360 뷰어 전체 검은 화면 (Breakdown)
- **분류**: 파노라마/씬 > 이미지 업로드 > 초소형 이미지 렌더링
- **재현 절차**
  1. 스페이스 에딧모드에서 100×50px 파노라마 이미지를 업로드
  2. 해당 씬을 360 뷰어에서 열고 드래그로 회전
- **기대**: 최소 해상도 미달 시 업로드 거부·안내, 또는 정상 렌더링
- **실제**: 업로드가 오류·경고 없이 허용되나 360 뷰어가 **완전 검은 화면**(회전해도 계속 검정, 사용자용 오류 메시지 없음). 씬이 1개뿐인 VT(`크래시_재현_단일`, 토큰 `152cb2925f12cffa1`)에서는 **투어 전체가 표시 불가**.
- **대조군(정상)**: 같은 스페이스의 `07_포맷.png`(정상 규격)는 정상 렌더링 → 초소형/저해상 입력에 대한 방어 부재가 원인.
- **근본원인 추정**: 업로드 시 최소 해상도/유효 파노라마(2:1 및 최소 픽셀) 검증 부재 + WebGL 텍스처 로드 실패 시 폴백/오류표시 없음.
- **증적**: 이미지 No.1 (검은 화면 캡처)

## 보통 (Major)

### E-002 — 잘못된/존재하지 않는 VT ID 접근 시 관리자 저작 화면 노출 (오류처리 부재)
- **분류**: 뷰어 > 오류 처리 > 잘못된 VT ID 접근
- **재현 절차**
  1. 존재하지 않는 주소 `https://roundpic.io/#/vt/ZZZZINVALID9999` 로 접속
  2. 새로고침(콜드 로드)
- **기대**: 404 / "존재하지 않는 투어" 등 오류 안내 페이지
- **실제**: 오류 페이지 없이 **관리자 저작 빈 화면**("등록된 360 이미지가 없습니다 / 우측 사이드바에서 이미지를 등록해주세요")과 **"360이미지 등록" 버튼**을 익명(비로그인) 사용자에게 노출.
- **영향**: 잘못된 입력에 대한 예외 처리 미흡 + 관리 UI 문구·버튼이 외부에 노출(정보 노출).
- **증적**: 이미지 No.2

## 경미 (Minor)

### E-003 — 원본 이미지 파일명 노출 및 브라우저 탭 title 미갱신
- **분류**: 파노라마/씬 > 파일명 노출 / 뷰어 > 브라우저 탭
- **재현**: 임의 VT를 뷰어로 열람 → 상단 제목, 씬 목록, 브라우저 탭 제목 확인. 씬을 전환하며 탭 제목 관찰.
- **기대**: 사용자 친화적 명칭(사업장/VT명) 표시, 파일명 미노출, 씬 전환 시 title 갱신
- **실제**: 원본 파일명(`로비 중앙.jpg`, `X3_H(1).jpg`, `06_초소형_100x50.jpg` 등 확장자 포함)이 상단 제목·씬 목록·**브라우저 탭 title**에 그대로 노출. 탭 title은 최초 씬 파일명으로 고정되어 씬 전환 시 갱신되지 않음.
- **영향**: UX/브랜딩 저하, SEO·북마크 부적절, 내부 파일명 노출.
- **증적**: 이미지 No.5

### E-004 — 로그인 이메일 필드 형식/필수 검증 미비
- **분류**: 계정/인증 > 로그인
- **재현**: (a) 이메일에 `notanemail`(@ 없음) + 비밀번호 `123456` 입력 후 로그인. (b) 이메일·비밀번호 빈 값으로 로그인.
- **기대**: (a) 이메일 형식 오류 안내·제출 차단, (b) 이메일·비밀번호 각각 필수 오류
- **실제**: (a) 형식 오류 없이 **유효(ng-valid)** 처리(형식 미검증), (b) 비밀번호만 오류 표시되고 **필수(*)인 이메일은 오류 미표시**(검증 불일치).
- **증적**: 이미지 No.3

### E-005 — 유효한 비밀번호 입력 후에도 오류 문구 잔존 (stale 검증 표시)
- **분류**: 계정/인증 > 로그인 > 비밀번호 검증 상태
- **재현**: 빈 제출로 비밀번호 오류 유발 → 비밀번호란에 유효한 `123456`(6자) 입력
- **기대**: 유효 입력 시 오류 문구 즉시 제거
- **실제**: 필드는 **유효(ng-valid)**로 판정되나 "비밀번호는 6자 이상 21자 미만입니다" 오류 문구가 빨간색으로 **잔존**.
- **증적**: 이미지 No.4

### E-006 — 스페이스 홍보링크 URL 형식 미검증
- **분류**: 스페이스 관리 > 스페이스 생성
- **재현**: [스페이스 등록] → 이름 입력 → 홍보링크란에 `not_a_valid_url_링크` 입력 → 저장
- **기대**: URL 형식 검증 후 안내 또는 정규화
- **실제**: 검증 없이 임의 문자열이 그대로 저장됨('스페이스가 등록되었습니다'). 뷰어에서 링크로 노출 시 오동작 소지.

### E-008 — 콘솔 런타임 에러 반복 발생
- **분류**: 비기능 > 안정성
- **재현**: 뷰어·에디터 조작 중 개발자도구 콘솔 확인
- **실제**: `TypeError: Cannot read properties of null (reading 'children') at getVisibleChildId` 가 뷰어·에디터 공통으로 반복 발생(사용자 화면에는 즉시 드러나지 않으나 잠재 결함 근거).
- **증적**: 이미지 No.7

### E-009 — 문서 언어 속성 오설정 (`<html lang="en">`)
- **분류**: 비기능 > 접근성/i18n
- **재현**: 페이지의 `<html lang>` 확인
- **실제**: 콘텐츠는 한국어인데 `lang="en"` → 스크린리더 발음·자동번역 부정확.

### E-011 — 에딧모드 진입 아이콘 간헐적 무반응 ※재현 불안정
- **분류**: 스페이스 관리 > 에딧모드 진입
- **재현**: 스페이스 카드의 에딧모드(이미지) 아이콘 클릭
- **실제**: 동일 조작에도 간헐적으로 반응 없음. **썸네일 더블클릭 시에는 정상 진입** → 진입 경로 신뢰성 저하.
- **주의**: 재현이 일정하지 않음. 제출 전 팀이 여러 브라우저/재시도로 **수동 재확인** 후 채택 여부 결정 권장.

## 개선사항

### E-007 — 변경 없이 취소해도 확인창 표시
- 스페이스 [정보수정] 진입 후 아무 것도 바꾸지 않고 [취소] → "취소시 작성 중인 내용이 삭제됩니다" 확인창이 불필요하게 표시. (증적 이미지 No.6)

### E-010 — Google Maps API 비동기 로딩 미적용
- 뷰어 로드 시 콘솔 경고: Maps JS API가 `loading=async` 없이 로드되어 성능 저하 가능.
