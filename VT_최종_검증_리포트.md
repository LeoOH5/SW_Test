> 2026-07-19 · 대상 VT partners(roundpic.io / vtapi.visitinc.kr / upload.visitinc.kr)
> 목적: v4 병합안의 결함이 **지금도 실제로 재현되는지** 자체 검증. 단일 세션·안전(active=false 유지, A-01 씬 생성 즉시 삭제, 병렬 로그인 금지).

자동/API 결함 중 **7건이 신선 증거로 재현 확정**(심각 1·보통 4·경미 2), A-02는 v4보다 **증거가 더 강해짐**, 4건은 이번 미실행(기존 증거만), 수동/UI E-series는 1건(E-014)이 자동 교차확인·나머지는 육안 재검증 필요.

### PASS(정상 동작) 재확인
| 항목 | 결과 |
|---|---|
| 비인증 API 차단 | `/content/list` 토큰 없이 → **401** `token is not valid` (정상) |

## 3. 정직성 노트
- 이번 실행에서 **실제 관측한 것만** "재현"으로 표기. 미수행(A-05/06/07/12)은 "결함 없음"이 아니라 **이번에 재확인 안 함**.
- A-01 씬은 생성 직후 삭제(남은씬 null), 최종 3개 스페이스 active 전부 false로 안전수칙 준수.
- 검증은 fable 에이전트 교차판정을 시도했으나, 본 판정은 **직접 수집한 신선 원자료(recon/verify_all.json)** 기준으로 확정.

## 4. 재검증으로 강화된 제출 포인트
1. **A-01**(심각) — 재현+ 확정. 개발사 우선 보고. 부가로 E-014(원본 파일명 노출)까지 같은 응답에서 확인.
2. **A-02**(보통) — code 101/102 구분 = 사용자 열거 근거를 코드 레벨로 강화. xlsx에 코드 명시 권장.
3. A-03·A-04 — 재현 확정, 그대로 제출 유효.
