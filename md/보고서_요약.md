# 결과보고서 요약 (제출 시 보고서 상단/별첨에 포함)

> 제품1 시트 서식은 변경 불가이므로, 이 요약은 **보고서 앞 별도 안내** 또는 제출 메일 본문/별첨으로 포함 권장.

## 테스트 개요
| 항목 | 내용 |
|---|---|
| 대상 제품 | **VT partners** (roundpic.io) — 360 VR 가상투어 저작·배포 (PC web) |
| 테스트 범위 | 관리자(회사·스페이스·360이미지·씬·핫스팟), 뷰어, 비기능(보안·안정성) |
| 테스트 계정 | 테스트 전용 계정 · **스타터 플랜** (회사 최대 10개, 스페이스 active 비활성 조건 준수) |
| 테스트 환경 | **Chrome / Windows 10** · 데스크톱 |
| 테스트 기간 | 2026-07-13 ~ (진행 중) |
| 테스트 데이터 | 개발사 제공 360 파노라마(F동·H동), 5MB 미만 리사이즈본 + 결함 유발용 변형 이미지 15종 |

## 핵심 결함 (심각도 순)
1. **E-008 (심각)** — 변형 이미지 배치 업로드 중 에디터 white screen 크래시 (새로고침 필요). 간헐 발생.
2. **E-009 (보통)** — 100×50 초소형 이미지가 씬으로 통과 → 뷰어 완전 검은 화면(검증·안내 부재).
3. **E-010 (보통)** — 안내는 "5MB까지"인데 7.4MB 통과, 35MB는 거부 → 용량 상한 비일관.
4. **E-012 (보통)** — 업로드 직후 에디터가 "이미지 없음"으로 잘못 표시, 새로고침해야 반영.
5. 경미 다수 — 안내문구 오류·피드백 부재·카테고리 명명 불일치·파일명 노출 등.

## 적용 테스트 기법
- **공식**: 경계값 분석(용량·해상도), 동등분할(파일 유형), 결정 테이블(접근 권한), 상태 전이(스페이스/씬).
- **창의**: API 직접 주입(UI 우회 보안 테스트), 변형 이미지 퍼징+SBTM, 페어와이즈 조합. → 상세 [기술성_산출물.md](기술성_산출물.md)

## 비고
- 결함 여부·심각도는 개발사 상의 후 최종 판정 대상 (공고 유의사항). 각 결함에 재현 절차·증적 첨부.
- 잔여: IDOR(수평 권한) UI 재시도, 업로드 용량 상한 이진탐색, 씬 삭제 정합성, 결함 증적 스크린샷.
