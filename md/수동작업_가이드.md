# 수동 작업 가이드 — 직접 해야 하는 것 & 파노라마 업로드 방법

> 대상: VT partners (roundpic.io Admin) · 작성 2026-07-13
> Claude가 자동으로 못 하는 작업만 모았습니다. 이 부분만 사람이 해주면 나머지 테스트는 이어서 진행됩니다.

---

## A. 왜 이 작업들은 직접 해야 하나 (제약 요약)

| 작업 | 자동화 불가 이유 |
|---|---|
| **로그인(비밀번호 입력)** | 비밀번호를 폼에 직접 입력하는 것은 안전 규칙상 Claude가 수행 불가 (테스트 계정이라도 동일). 이메일까지는 자동 입력 가능. |
| **파노라마 업로드** | 드라이브 파노라마가 **장당 34~37MB**인데 자동 업로드 도구는 **호출당 10MB 상한**. + OS 파일 선택창은 자동 조작 불가. |
| **파일 다운로드** | 파일 다운로드는 사용자 확인이 필요한 작업. 드라이브 → 로컬 다운로드는 직접. |
| **서류 제출·이메일 발송** | 대회 접수 메일 발송, 서류 제출 등 대외 발송은 직접(발송 전 내용은 Claude가 준비 가능). |

> 위 4가지만 사람이 하고, **UI 탐색·기능 클릭·폼 입력(비번 제외)·결함 재현·보고서 작성**은 Claude가 진행합니다.

---

## B. 직접 해야 하는 것 체크리스트

- [ ] **로그인**: 로그인 화면에서 비밀번호 `[비밀번호는 test_rule.md 참조(커밋 제외)]` 입력 후 [로그인] 클릭 (이메일은 자동 입력됨)
- [ ] **파노라마 업로드**: 아래 C 절차대로 (1장 → 3~4장)
- [ ] (D2 이후) **비정상 이미지가 필요할 때**: 변형본 파일을 만들면 Claude가 안내 — 업로드는 직접
- [ ] (마감 주) **결과보고서·서류 최종 제출**: `ljs@sjtp.or.kr` 로 메일 발송은 직접

---

## C. 파노라마 업로드 방법 (핵심)

### 준비: 브라우저 탭 2개
- **에디터 탭**: roundpic 편집 화면 (스페이스 카드 클릭 시 진입, URL `#/vt/...`)
- **드라이브 탭**: 구글 드라이브 `sw경진대회` 폴더 (파노라마 원본 위치)

### 업로드 순서 — "1장 먼저 → 3~4장 추가"

**1단계) 이미지 1장 다운로드**
1. 드라이브 탭에서 파노라마 1장 우클릭 → **다운로드** (예: `X3_F-1(2).jpg`)
2. 로컬(다운로드 폴더)에 저장 확인

**2단계) 에디터에서 업로드**
1. 에디터 탭에서 우측 **`◀` 화살표**로 사이드바 펼치기
2. 사이드바 상단 **`+` 타일** 또는 **`360이미지 등록`** 클릭
3. 파일 선택창에서 방금 받은 이미지 선택 → 열기
4. 360 씬이 화면에 뜨면 **1장 업로드 완료** → Claude에게 "1장 올렸어" 알려주기

**3단계) 3~4장 추가 (멀티씬 구성)**
1. 같은 장소 세트를 추천 — 예: `X3_H-meetingroom(1)` ~ `(4)` 4장
2. 드라이브에서 여러 장 선택 후 다운로드 → 에디터에서 다시 `360이미지 등록`으로 업로드
   (파일 선택창에서 **여러 장 동시 선택**하면 다중 업로드도 함께 테스트됨)
3. 씬이 여러 개 뜨면 "여러 장 올렸어" 알려주기 → Claude가 씬/핫스팟 테스트 진행

### 왜 1장 → 3~4장인가
- **1장**: 다른 변수 없는 "정상 단일 업로드" PASS 기준선(깨끗한 증적)
- **3~4장**: 씬 이동 핫스팟, "대상 씬 삭제 → 깨진 링크", 씬 목록/순서/카테고리 탭, 다중 업로드 — 씬이 여러 개여야 되는 테스트들 때문
- **전부(약 15장)는 불필요**: 커버리지 안 늘고 업로드만 오래 걸림
- **비정상 이미지(초대용량·0바이트·비율·포맷)는 이 정상 파일이 아니라 따로 변형본을 만들어** 사용 → 정상 파일은 몇 장만

---

## D. 업로드할 때 지킬 것 (제약)

- ⚠ **스페이스 active 토글은 OFF 유지** (test_rule 지시 — active 켜면 외부 노출/운영 영향)
- ⚠ **회사(company)는 최대 10개** — 회사는 새로 만들지 말고 기존 `SW테스트_회사01` 재사용
- ⚠ 계정은 **테스트 용도로만** — 파괴적 시도(대량요청·실제 익스플로잇) 금지
- 정상 규격 파노라마는 **2:1 equirectangular** (드라이브 원본이 규격에 맞음)
