# 결함 증적 캡처 체크리스트

> 각 결함을 재현해 스크린샷을 찍고 **`증적/E-xxx.jpg`** 로 저장 → 결과보고서 「제품1_관련이미지」 시트에 이미지No–결함ID 매핑(이미 매핑 완료).
> 캡처는 화면 앞 담당자가 수행. 아래 "캡처 화면"이 결함이 드러나는 순간.

| 결함ID | 심각도 | 재현 위치·절차 | 캡처 화면(무엇이 보여야 하나) |
|---|---|---|---|
| **E-008** | 심각 | 빈 스페이스 에딧모드 > 변형 이미지 여러 장 배치 업로드 | **white screen(하얀 빈 화면)** + DevTools 콘솔의 `null (reading 'children')` |
| **E-009** | 보통 | 회사2 > `크래시_재현_단일` 스페이스 진입(새로고침) | 씬 `06_초소형_100x50.jpg`가 **완전 검은 화면**으로 렌더 |
| **E-010** | 보통 | ①원본 35MB 업로드 시 "5MB까지" 거부 안내 / ②`업로드_테스트` 씬목록의 `12_경계_5MB초과.jpg`(7.4MB) 통과 | 거부 안내 + 7.4MB 씬이 목록에 있는 두 장면 |
| **E-012** | 보통 | 빈 스페이스에 이미지 1장 업로드 직후(새로고침 전) | 헤더가 "스페이스명 360이미지명" 플레이스홀더 + "이미지 없음" 표시 |
| **E-011** | 경미 | 회사2 > `업로드_테스트(변형이미지)` 에딧모드 사이드바 상단 | **"테스트" 카테고리 탭 2개** + 하나 클릭 시 "360이미지가 없습니다"(빈 유령) |
| **E-014** | 경미 | 아무 스페이스 에딧모드/뷰어 | 상단 헤더에 **원본 파일명(X3_H-1(1).jpg 등) 노출** |
| **E-001** | 개선 | 스페이스 [수정](연필) > 값 변경 없이 [취소] | **"취소시 작성 중인 내용이 삭제됩니다"** 경고 팝업 |
| **E-002** | 경미 | 스페이스 등록 완료 순간 | 토스트 **"스페이스가 등록 되었습니다"**(띄어쓰기) |
| **E-003** | 경미 | 에딧모드 `?` 도움말 화면 + 파노라마 우클릭 메뉴 | 도움말 "정보·이미지·영상·링크"(4) vs 메뉴 "공지사항·이미지·비디오·링크·예약"(5) 두 장 |
| **E-005** | 경미 | `테스트_스페이스01`(카테고리 X3_F-1) + `H동 회의실&구역`(카테고리 회의실&구역) | 두 스페이스 사이드바 카테고리명 대비 |
| **E-006** | 경미 | 에딧모드 `공유` 클릭 후 | 클릭해도 변화 없는 화면(토스트/모달 없음) — 필요시 영상 권장 |
| **E-007** | 경미 | 스페이스 등록 후 [스페이스 등록] 재오픈 | 폼에 직전 입력값 잔존 |
| **E-013** | 경미 | 예약 핫스팟 > 구역 미선택 [저장] | "예약 버튼 연결" 팝업 그대로(저장·안내 없음) |
| **E-015** | 경미 | 빈/일반 에딧모드에서 DevTools 콘솔 | `getVisibleChildId ... null children` 반복 에러 로그 |

> 팁: 콘솔 캡처가 필요한 결함(E-008/E-015)은 F12 > Console 열고 함께 캡처. 심각·보통(E-008/009/010/012)부터 우선 확보.
