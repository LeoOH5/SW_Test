## 1. 확정한 실제 기능트리 (실물 UI 기준) — ⚠ 계획서와 계층 다름

기존 계획은 "프로젝트(VT)" 단일 계층으로 가정했으나, **실제는 3계층**입니다:

```
[전역 상단바]
  회사 관리 · 플랜 · 내 정보 · Contact · 계정메뉴(내정보/로그아웃/개인정보처리방침/서비스이용약관)

[계층 구조]  회사(Company) → 스페이스(Space) → 360이미지/씬(에딧모드)

1. 계정/인증
   1.1 로그인 (#/login)            1.2 비밀번호 찾기
   1.3 로그아웃 / 세션              1.4 솔루션 이용 문의
   1.5 내 정보(이름·전화번호 변경 / 이메일 변경문의 / 비밀번호 변경 / 소속회사)
   1.6 플랜(스타터[사용중]·라이트·프로·엔터프라이즈, 결제=네이버페이 안내)
2. 회사(Company) 관리   ※ 최대 10개
   2.1 등록(회사명*·회사이메일*·대표자명*·전화번호·주소)  2.2 목록(멤버·일간방문자)
   2.3 수정  2.4 삭제
3. 스페이스(Space)
   3.1 등록(스페이스명*·홍보링크·메모)  3.2 목록  3.3 수정  3.4 삭제
   3.5 active 토글 (⚠ 비활성화 유지)   3.6 뷰어 프리뷰(이미지 아이콘)
4. 에딧모드(360 에디터, #/vt/{id}?cntId={n})
   4.1 360이미지 업로드(우측 사이드바 `업로드`/`+`, input type=file) — 씬으로 등록
   4.2 씬 목록: 카테고리 자동그룹(파일명 접두사), 씬별 ★대표설정·연필(타이틀 변경)·휴지통(삭제)
   4.3 핫스팟: **우클릭 → 유형 선택 → 위치 클릭**. 유형 5종 = **공지사항·이미지·비디오·링크·예약**
        (링크 = 외부 URL 연결[제목+URL] / 씬이동 전용 유형은 없음 → 씬 전환은 씬목록 패널 기반으로 추정)
   4.4 상단 지도·공유, 하단 관리자/뷰어 모드 토글, 리스트보기(핫스팟 목록)
5. 뷰어(공개) / 6. 비기능(호환·성능·콘솔·보안)
```

> 계정 → 회사 → 스페이스 → 에딧모드 → 멀티씬 업로드 → 핫스팟 생성까지 직접 확인 완료.

### 보안(권한) 테스트 결과 — D1
- **무단 접근 차단 확인(PASS)**: 투어 데이터 API `vtapi.visitinc.kr/content/auth?cntId=149`를 토큰 없이 호출 → **401 "token is not valid"**. active OFF 스페이스 데이터가 인증 없이 노출되지 않음. `/user`도 401.
- 인증 방식: **헤더(Bearer 토큰) 기반**(쿠키 아님 — credentials:'include'는 CORS 차단). → "1번 무단 URL 접근" 관점의 **심각 결함 없음**(접근제어 정상). 보안 테스트 수행 근거로 보고서에 PASS 기록.

## 10. 승산 굳히기 세션 (용량 상한 확정 + 업로드 아웃티지)

### ✅ E-010 대폭 강화 (보통, 확정)
- 용량 사다리 업로드 결과: **`25MB.jpg`·`30MB.jpg`(실제 각 24MB)가 씬으로 통과**(사이드바 등록 확인). `06MB.jpg`도 통과.
- 즉 **실제 업로드 상한은 최소 24MB 이상**(35MB는 거부) — 그런데 안내는 "5MB까지"였음.
- → **안내(5MB)와 실제 상한(24~35MB)이 5배+ 불일치.** 보고서 E-010에 반영(재현절차: 24MB 통과 / 35MB 거부 / 안내 5MB).

### ⚠ 업로드 전면 실패 발생 (요확인 — 재현/원인 미확정)
- 세션 후반 **정상 2:1 파일(08MB)조차 '업로드 실패'** — 단일·다중 모두. 아침엔 정상이었음.
- 콘솔: 업로드는 비동기 job(`[sceneUpload] jobId=...`). GET `/category`는 200(세션 유효).
- 원인 후보: **세션/토큰 만료, 업로드 job 서비스 일시 불안정, 세션 과부하**. → **제품 결함으로 단정 불가**.
