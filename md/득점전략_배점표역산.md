## 항목 1. 결함 발견 (50점) — 심각도 가중 최대화

### 채점 구조 해부
- 점수 = Σ(각 결함의 심각도별 배점), **상한 50점**.
- 심각도(양식 정의): **심각(Critical)** = 기능 진행 불가 Breakdown / **보통(Major)** = 동작하나 명세와 다름 / **경미(Minor)** = 사소·개선.
- **결함 여부와 심각도는 개발사와 상의 후 최종 판정** → 우리가 "심각"이라 우겨도 깎이면 그 배점이 날아감.

### 만점 도달 설계 (역산)
심각 배점을 정확히는 모르지만, 심각>보통>경미 가중이 확실하므로:

| 전략 | 내용 |
|---|---|
| **심각·보통 결함을 우선 확보** | 상한 50을 채우려면 고배점 결함 **6~10건**을 확실히 잡는 게 경미 30건보다 빠름 |
| **결함을 기능 영역에 분산** | 한 기능에서 파생된 결함은 "동일 원인 중복"으로 묶여 1건 처리될 수 있음 → **서로 다른 근본원인**의 결함으로 폭을 넓혀 총합 saturate |
| **판정 리스크 최소화** | 각 결함마다 ① 명확한 재현절차 ② 증적(스크린샷/영상/콘솔로그) ③ "명세상 이래야 하는데 이렇더라"의 **기대-실제 대비**를 붙여 개발사 반박 여지를 없앰 |
| **과대 심각도 자제** | 경미를 심각으로 부풀리면 판정에서 강등 → 신뢰 하락. **정직하게 등급 매기되 근거로 방어** |

### 고배점 결함이 나오는 지점 (심각도 높은 순)
1. **인증·권한 (심각 유력)** — 로그아웃 후 뒤로가기로 관리화면 접근 / **뷰어·관리 URL의 VT id 변경으로 남의 비공개 VT 접근(IDOR, 수평 권한상승)** / 세션 만료 후 저장. → 보안성 결함은 심각 판정 받기 쉬움.
2. **파노라마 업로드 Breakdown (심각)** — 초대용량·0바이트·손상 파일로 업로드/저장이 **아예 실패하고 복구 불가** / 업로드 중 이탈 시 데이터 유실.
3. **저장·삭제 정합성 (보통~심각)** — 삭제한 VT가 목록/링크에 잔존 / 이동 핫스팟 대상 씬 삭제 시 깨진 링크 / 두 탭 동시편집 충돌.
4. **핫스팟 동작 오류 (보통)** — 예약 핫스팟에 과거 날짜 허용 / 핫스팟이 엉뚱한 씬으로 이동 / 사진·동영상 핫스팟 미재생.
5. **입력검증·UI (경미~보통)** — 파일명 원본 노출(프라이버시), 글자수·이모지·공백 처리, 해상도별 레이아웃 깨짐, 콘솔 에러.

> **운영**: 2인 중 A=조작·공격, B=즉시 기록·캡처. 발견 즉시 재현 재실행으로 확정 후 결함ID 부여. **정상 규격 이미지(개발사 제공)로 PASS 기준선을 먼저 잡고**, 경계·이상 이미지로 결함을 유발.

## 항목 2. 테스트 커버리지 (10점) — "빠짐없이"를 증명

채점 포인트: **대상 SW 기능 전체를 얼마나 덮었는가.** → 결함 유무와 무관하게, **모든 기능에 TC가 존재**하고 **PASS 케이스까지 기록**돼야 만점.

### 만점 체크리스트
- [ ] **기능 인벤토리 표**를 보고서에 명시 (아래 트리) → "우리가 인식한 전체 기능 = N개"를 먼저 선언
- [ ] N개 기능 **각각에 최소 1개 TC** 매핑, 커버 여부 표시 → 커버리지 % 산출해서 제시
- [ ] **PASS 케이스 전부 기록** (FAIL만 적으면 "안 건드린 기능"으로 보여 커버리지 손해)
- [ ] 정상 흐름 + 예외 흐름 + 경계 모두 포함

### 기능 인벤토리 (커버리지 근거 · 실제 서비스 확인 기반)
```
1. 계정/인증 : 로그인 · 비밀번호찾기 · 로그아웃 · 세션 · 문의
2. 프로젝트(VT) : 생성 · 목록 · 수정 · 삭제 · 복제
3. 파노라마/씬 : 업로드 · 씬생성/삭제 · 순서/이름 · 카테고리 · 지도좌표
4. 핫스팟 : 씬이동 · 사진 · 동영상 · 예약
5. 뷰어 : 360회전/줌 · 씬이동 · 씬목록패널/탭 · 지도토글 · 공유/링크
6. 비기능 : 브라우저호환 · 반응형 · 성능 · 콘솔에러 · 보안
```
> 뷰어(로그인 불필요) 기능은 **계정 없이도 지금 커버 시작 가능** → 커버리지 조기 확보.

## 항목 3. 테스트 케이스 완성도 (10점) — TC 한 줄 한 줄의 품질

채점 포인트: **설계한 TC의 완성도(질).** 양식의 각 열을 빠짐없이, 재현 가능하게 채웠는가.

### 만점 TC의 요건 (모든 행에 적용)
| 열 | 완성 기준 |
|---|---|
| 대/중/소분류 | 기능 인벤토리와 일치 |
| 테스트 절차 | **그대로 따라 하면 재현되는** 번호 매김 (전제조건 포함) |
| 기대 결과 | 명세/상식 기반 **구체적 서술** ("정상 처리" 금지, 무엇이 어떻게) |
| 실제 결과 | 관찰한 그대로, 스크린샷 대응 |
| PASS/FAIL | 명확 |
| 심각도 | FAIL만, 양식 정의대로 |
| 결함ID | 1결함 1ID, 순차 |
| (추가 권장) 적용 기법 | 각 TC에 기법 태그 → 기술성 점수와 연동 |

> **팁**: 기대결과가 모호한 TC는 감점 요인. "로그인 됨"이 아니라 "메인(PROJECT LIST)으로 이동하고 상단이 Login→로그아웃으로 바뀜"처럼 검증 가능하게.

## 항목 5. 공식 테스트 기법 적용 (10점) — 이름값을 증명

채점 포인트: **구조/명세/경험 기반 등 공식 기법을 실제로 적용**했는가. 태그만 달지 말고 **설계 과정(표·다이어그램)을 산출물로 보여야** 만점.

### 적용할 공식 기법 (블랙박스 중심) + 증거물
| 기법 | 적용 대상 | 보고서에 넣을 증거 |
|---|---|---|
| 동등분할 | 로그인 ID/PW, 업로드 파일 유형 | 유효/무효 클래스 분할표 |
| **경계값 분석** | 업로드 용량 상·하한, 제목 글자수, 예약 날짜 | 경계값 도출표 (min-1, min, max, max+1) |
| **결정 테이블** | (로그인여부 × VT공개상태 × 소유자) → 접근 | 조건-액션 결정표 |
| **상태 전이** | VT: 작성중→발행→수정→삭제 / 세션 만료 | 상태전이 다이어그램 + 전이표 |
| 유스케이스 기반 | "VT 하나를 만들어 배포"까지 end-to-end | 시나리오 흐름 |
| 경험 기반(오류추정) | 과거경험상 취약지점 체크리스트 | 오류추정 체크리스트 |

> **만점 팁**: 최소 **3개 기법을 "설계 산출물"과 함께** 제시(경계값표·결정표·상태전이도). 말로만 "적용했다"는 감점.

## 항목 6. 창의적 기법 사용 (10점) — 차별화 1~2개를 깊이 있게

채점 포인트: 계획·설계·구현에 **창의적 테스트 기법**을 썼는가. 흔한 기법 나열보다 **1~2개를 왜/어떻게까지** 풀어야 함.

### 추천 창의 기법 (근거까지 서술)
1. **페어와이즈(Pairwise) 조합 최적화** — 브라우저(3)×해상도(3)×핫스팟종류(3)=27조합을 페어와이즈로 **9케이스로 축소**, "적은 케이스로 결함검출력 유지" 논리 + 조합표 첨부.
2. **네트워크 조건 주입 테스트** — Chrome DevTools로 오프라인/Slow 3G/스로틀링에서 대용량 파노라마 로딩·업로드 → **실사용 회선 재현**, 일반 테스트가 놓치는 결함.
3. **SBTM(세션 기반 탐색적 테스트)** — 90분 차터("업로드를 30분간 파괴") 운영, **세션노트·발견결함 로그를 부록으로 첨부** → 정성 평가에 강함.

> 3개 중 **2개를 골라 각각 반 페이지씩** 방법·발견·판단을 서술하면 만점권.

## 항목 7. 가산점 — 시험장 밖에서 따는 확정 점수

| 항목 | 점수 | 액션 (마감 7/20 전) |
|---|---:|---|
| **SW 관련 학과 재학** | 2 | 본인 재학증명서 발급 (**확정 2점**). 팀원도 SW과면 서류 제출 |
| **교육 수료증** | 1/장 | 배움나라·K-MOOC·NIPA 이러닝 등 **무료 SW/품질/테스팅 강의** 각자 2~3장 수료 → 2~3점 |
| **SW테스팅 자격증** | 2/장 | ISTQB/KSTQB/CSTS. 예선 마감 전 취득은 일정상 어려우나 **8월 KSTQB Foundation** 응시 계획(본선 대비 겸용) |

> 가산점은 상한 표기가 없으니 **수료증을 최대한 모으면 순수 가산.** 온라인 무료 강의 수료가 12일이면 충분 → **가장 효율 좋은 점수원.**

## 종합 — 목표 점수 배분 (현실적 만점 시나리오)

| 항목 | 목표 | 확보 전략 요약 |
|---|---:|---|
| 결함 발견 | 45~50 | 심각·보통 결함 6~10건 확정, 증적·재현으로 판정 방어 |
| 커버리지 | 10 | 기능 인벤토리 전 기능 커버, PASS 포함 |
| TC 완성도 | 10 | 재현 가능·기대결과 구체·기법 태그 |
| 보고서 완성도 | 10 | 양식 100% 준수 + 요약·환경·증적 매핑 |
| 공식 기법 | 9~10 | 경계값표·결정표·상태전이도 등 **설계 산출물** 제시 |
| 창의 기법 | 9~10 | 페어와이즈/네트워크주입/SBTM 중 2개 심화 |
| **본점** | **93~100** | |
| 가산점 | +4~6 | 재학증명 2 + 수료증 2~3 |

> **결론**: 완성도 30점 + 기술성 20점 + 가산점은 **노력만 하면 거의 확정으로 채워지는 50여 점**이다. 여기서 만점을 다지고, 변동성이 큰 결함 50점에 화력을 집중하면 세종 상위 2팀 안착은 물론 본선 수상권까지 노릴 수 있다. **완성도·기법·가산점을 먼저 굳히고, 결함에 올인하라.**
