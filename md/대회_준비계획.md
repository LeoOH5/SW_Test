## 2단계 — 제품 분석 + 테스트 설계 (7/9 ~ 7/13)

### 기능 트리 만들기 (대분류 → 중분류 → 소분류)
보고서 양식의 분류 체계이자 **커버리지 10점의 근거**. 예상 기능 축:

- **계정**: 가입 / 로그인 / 비밀번호 재설정 / 권한
- **VT 생성**: 파노라마 업로드 / 씬 구성 / 씬 간 이동 연결
- **핫스팟**: 사진 / 동영상 / **예약** (인터랙션 3종)
- **VT 관리**: 수정 / 삭제 / 복제 / 목록
- **공유·배포**: 링크 / 임베드 / 뷰어 화면
- **뷰어 UX**: 360 회전 / 줌 / 모바일·브라우저 호환

### 테스트 케이스 설계 — 각 TC에 적용 기법 명시 (기술성 10점)
**명세 기반**
- 동등분할, 경계값 (업로드 용량·글자수·예약 날짜)
- 결정 테이블 (권한 × 상태 조합)
- 상태전이 (VT 작성중 → 발행 → 삭제)

**경험 기반**
- 탐색적 테스트 (세션 차터 기록), 오류 추정

**창의적 기법 10점용 (1~2개 준비, "왜 이 기법인지" 서술)**
- 페어와이즈 조합 (브라우저 × 해상도 × 핫스팟 종류)
- 개발자도구 네트워크 오프라인/스로틀링 테스트
- 세션 기반 탐색적 테스트(SBTM) 차터 운영

## 3단계 — 집중 테스트 (7/12 ~ 7/17) · 50점짜리 결함 사냥

### 웹 VR 저작도구에서 결함이 잘 나오는 곳
1. **업로드 경계**: 초대용량/0바이트 파노라마, 지원 외 포맷, 한글·특수문자·긴 파일명, 업로드 중 페이지 이탈
2. **인증·권한**: 로그아웃 후 뒤로가기, URL의 ID 변경으로 남의 VT 접근(수평 권한 상승), 세션 만료 후 저장 시도
3. **예약 핫스팟**: 과거 날짜 예약, 자정·월말 경계, 동시 예약
4. **상태 꼬임**: 편집 중인 씬 삭제, 삭제된 VT의 공유 링크 접속, 두 탭 동시 편집
5. **입력값**: 최대 길이, 이모지, HTML 태그(`<b>` 정도만 — 파괴적 시도 금지)
6. **UI/뷰어**: 해상도별 레이아웃 깨짐, Chrome/Edge/Firefox 차이, 콘솔 에러

### 운영 방식
- 역할 분담: **한 명 조작·탐색, 한 명 즉시 기록·스크린샷**
- 결함 발견 즉시 재현 절차를 다시 밟아 검증 후 증적 확보
- **심각 결함 소수 > 사소 결함 다수** (심각도 배점이 곱해짐)
- 단, 사소(개선사항)도 커버리지·완성도 점수에 기여하니 전부 기록
- 결함 여부는 개발사와 상의 후 판정 → **재현 절차 명확 + 증적 확실한 것만** 보고

## 4단계 — 보고서 완성 (7/17 ~ 7/19, 늦어도 7/20 아침 제출)

### 양식 규칙 준수 = 점수
- **PASS 케이스도 전부 기록** (FAIL만 적으면 커버리지 10점 포기)
- 결함 1건당 결함 ID 1개 (예: E-001), 증적 이미지 시트에 ID 매핑
- 테스트 절차는 "그대로 따라 하면 재현되는" 수준으로 번호 매겨 작성
- 표 행 추가는 자유, **표 양식(서식) 변경 금지**
- 줄바꿈은 ALT + ENTER

### 제출 전 체크리스트
- ☐ SW 테스트 결과보고서 (xlsx)
- ☐ 참가신청서 (hwp)
- ☐ 개인정보 수집·이용 동의서 + 참가서약서
- ☐ 참가자격 증빙 (팀원 전원)
