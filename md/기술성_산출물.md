# 기술성 산출물 — 공식 기법(10) + 창의적 기법(10)

> 2026 중부권 SW테스트 경진대회 · 대상 **VT partners (roundpic.io)**
> 제출 방식: 이 표들을 **결과보고서 별첨** 또는 **제품1 시트 하단 참고**로 포함. 각 기법은 실제 수행한 테스트에 근거.
> 배점 근거: 배점표 「기술성 — 공식 테스트 기법 적용 10 / 창의적 기법 사용 10」

## A. 공식 테스트 기법 (10점)

### A-1. 경계값 분석(BVA) — 360이미지 업로드 파일 용량
업로드 허용 용량의 경계를 min/max 인접값으로 도출. (정확한 상한은 이진탐색으로 확정 예정)

| 구분 | 입력 파일 | 기대 결과 | 실제 결과 | 판정 |
|---|---|---|---|---|
| 하한 이하 | 0바이트 | 거부 | '업로드 실패' 거부 | PASS |
| 정상역 | 4.1MB (8192×4096) | 허용 | 정상 등록 | PASS |
| 경계 부근(안내값) | "5MB까지 가능" 안내 | 5MB 초과는 거부 | **7.4MB가 통과됨** | **FAIL (E-010)** |
| 초대용량 | 35MB 원본 | 거부 또는 허용(일관) | '5MB까지'로 거부 | 상한 비일관 |

> **결함 근거**: 안내(5MB)와 실제 수용(7.4MB 통과)이 불일치 → 상한 경계 미정의(E-010). **추가**: 6/8/10MB 이진탐색으로 실제 상한 특정 예정.

### A-2. 경계값 분석(BVA) — 이미지 해상도/비율
| 구분 | 입력 | 기대 | 실제 | 판정 |
|---|---|---|---|---|
| 최소 미만 | 100×50 | 거부/안내 | 통과 → 뷰어 검은화면 | **FAIL (E-009)** |
| 정상 2:1 | 8192×4096 | 정상 | 정상 | PASS |
| 비율 이탈 | 1:1 / 16:9 / 세로 | 검증/보정 | (개별 확정 예정) | 확인중 |

### A-3. 동등분할(Equivalence Partitioning) — 업로드 파일 유형
| 클래스 | 대표값 | 기대 |
|---|---|---|
| 유효 | 2:1 equirectangular JPG (≤허용용량) | 수용 |
| 무효-포맷 | png/webp/gif/bmp, 확장자 위장 | 거부/안내 |
| 무효-비율 | 1:1, 16:9, 세로 | 거부/보정 |
| 무효-용량 | 0바이트, 초대용량 | 거부 |
| 무효-손상 | 잘린/텍스트 위장 파일 | 거부 |

> 각 무효 클래스에서 대표값 1개씩 테스트 → 최소 케이스로 검출력 확보. 손상/0바이트/위장은 정상 거부(PASS), 초소형·초과용량은 결함(E-009/E-010).

### A-4. 결정 테이블(Decision Table) — 콘텐츠 접근 권한
조건 조합에 따른 접근 허용 여부. (실측: 인증 토큰 없으면 401 차단)

| 규칙 | 인증토큰 | 소유자여부 | active(발행) | → 접근 | 실측 |
|---|---|---|---|---|---|
| R1 | 없음 | - | OFF | 차단 | **401 차단 확인(PASS)** |
| R2 | 없음 | - | ON | 차단(뷰어는 공개?) | 미검증 |
| R3 | 있음(소유) | 예 | OFF | 허용 | 정상 편집(PASS) |
| R4 | 있음 | **아니오(타 cntId)** | - | **차단이어야** | **IDOR 미검증(추가)** |

> R1 실측: `content/auth?cntId=` 토큰 없이 호출 시 401 'token is not valid' → 무단 접근 차단 정상. R4(수평 권한상승)는 UI 기반으로 재시도 예정.

### A-5. 상태 전이(State Transition) — 스페이스/씬 생명주기
```
[스페이스] 생성 ──이미지 업로드──> 편집가능 ──active ON──> 발행(공개) ──active OFF──> 비공개
                                   │
                                   └─업로드 오류/크래시(E-008)─> white screen(비정상 종료)
[씬] 업로드 ──> (즉시 반영되어야) ──> 목록 표시
              └─실제: 빈 상태 표시, 새로고침 후 반영(E-012, 비정상 전이)
```
> 정상 전이 외에 **비정상 전이**(업로드→크래시 E-008, 업로드→빈상태 E-012)를 상태도로 포착.

## B. 창의적 테스트 기법 (10점) — 2개 심화

### B-1. API 직접 주입 테스트 (UI 우회 보안 테스트) ★
**왜**: UI만 테스트하면 접근제어 결함(비공개 데이터 노출)을 놓친다. 브라우저 콘솔에서 백엔드 API를 직접 호출해 **인증 계층을 우회 시도**.

**방법**: DevTools에서 `fetch('https://vtapi.visitinc.kr/content/auth?cntId=149', {credentials:'omit'})` — 토큰/쿠키 없이 비공개(active OFF) 스페이스 데이터 요청.

**발견**:
- 무단 호출 → **401 'token is not valid'** → 접근제어 정상(심각 결함 없음, PASS 근거 확보).
- 인증 방식이 헤더(Bearer) 기반임을 확인(쿠키 아님).

**판단**: 일반 UI 테스트로는 증명 불가한 "비공개 콘텐츠 보호 여부"를 **정성적·기술적으로 검증**. 접근제어가 견고함을 근거와 함께 보고.

### B-2. 변형 이미지 퍼징 + 세션 기반 탐색(SBTM) ★
**왜**: 업로드는 결함 밀집 지점. 정상 파일만으로는 검증 로직의 허점을 못 찾는다. **규격을 의도적으로 위반한 15종 변형 이미지**를 제작해 업로더를 퍼징.

**차터(90분 세션)**: "업로드 검증을 파괴한다" — 0바이트·손상·비율이탈(1:1/16:9/세로)·초소형(100×50)·지원외 포맷(png/webp/gif/bmp)·확장자 위장·용량 경계(7.4MB)·특수문자/이모지/초장문 파일명.

**발견(세션 로그)**:
| 입력 | 결과 | 결함 |
|---|---|---|
| 0바이트/손상/텍스트위장 | 거부 | (정상) |
| 100×50 초소형 | 통과 → 검은화면 | E-009 |
| 7.4MB | 통과(안내는 5MB) | E-010 |
| 배치 업로드 | white screen 크래시 | E-008 |
| 업로드 직후 | 빈 상태 표시 | E-012 |
| 카테고리 | 빈 유령 탭 생성 | E-011 |

**판단**: 탐색적 세션에서 **보통 3 + 심각 후보 1**을 도출. 정형 케이스가 놓치는 결함을 세션 노트로 증적화.

### B-3. (보조) 페어와이즈(Pairwise) 조합 최적화
**왜**: 브라우저×해상도×핫스팟유형 전조합은 3×3×5=45케이스. 페어와이즈로 **모든 2요소 쌍을 1회 이상 커버**하며 케이스 수 축소.

| # | 브라우저 | 해상도 | 핫스팟 유형 |
|---|---|---|---|
| 1 | Chrome | 1920 | 공지사항 |
| 2 | Chrome | 1366 | 비디오 |
| 3 | Chrome | 모바일 | 예약 |
| 4 | Edge | 1920 | 비디오 |
| 5 | Edge | 1366 | 예약 |
| 6 | Edge | 모바일 | 링크 |
| 7 | Firefox | 1920 | 예약 |
| 8 | Firefox | 1366 | 링크 |
| 9 | Firefox | 모바일 | 이미지 |

> 45 → **9케이스**로 축소하며 모든 쌍 커버 → "적은 케이스로 넓은 커버리지" 논리.

## C. TC ↔ 적용 기법 매핑 (보고서 연동)
| 기법 | 관련 TC / 결함 |
|---|---|
| 경계값 분석 | 업로드 용량(E-010), 해상도(E-009) |
| 동등분할 | 업로드 파일 유형(깨진파일 거부 PASS, E-009) |
| 결정 테이블 | 접근 권한(무단 401 PASS, IDOR 예정) |
| 상태 전이 | 업로드 크래시(E-008), 상태반영(E-012) |
| API 주입(창의) | 무단 접근 차단(PASS) |
| 퍼징+SBTM(창의) | E-008/009/010/011/012 |
| 페어와이즈(창의) | 호환성·핫스팟 조합 |

> **만점 논리**: 공식 4기법을 **설계 산출물(표·상태도)**과 함께 제시 + 창의 2기법을 **방법·발견·판단까지 서술**. 말로만이 아니라 실제 수행 근거로 방어.
