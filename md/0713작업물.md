- `회의실&구역\` : 리사이즈 6장 / `층별투어\` : 리사이즈 37장 (각 ~4MB, 8192×4096)

- (참고) `Downloads\X3_F-1(2)_5mb.jpg` 등 F 시리즈 리사이즈

**리사이즈 방법** (다른 환경에서 재현 시): Python + Pillow. 원본 2:1 유지, `8192×4096`, JPEG 품질 82~88 → 약 4MB. (원본 35MB는 안내상 5MB 제한 때문에 만들었으나 실측 상한은 24MB+라 사실 불필요했음 — E-010 참고)

## 9. 방법론 · 도구 (다음 사람이 같은 방식으로 이어가려면)

- **UI 테스트**: 브라우저 자동화(또는 수동)로 관리자 조작. 우클릭=핫스팟, 카드 본문 클릭=에딧모드 진입.
- **이미지 준비**: Python + Pillow. 원본 2:1 → 8192×4096 리사이즈(품질82~88, ~4MB). 변형(0바이트·손상·비율·포맷·용량)도 Pillow로 생성.
- **보고서**: openpyxl로 원본 양식 사본에 행 추가(표 서식 변경 금지, 병합 유지, PASS도 전부 기록, 결함ID는 XX-001부터, 줄바꿈 ALT+ENTER).
- **보안 테스트(창의)**: 콘솔에서 `fetch('vtapi.visitinc.kr/content/auth?cntId=', {credentials:'omit'})` → 401 확인(무단 접근 차단 PASS 근거).
- **디버깅**: 콘솔(`[sceneUpload]`·에러) + 네트워크 모니터링.

## 10. 핵심 교훈 / 주의점

- **업로드는 비동기 job이라 결과가 새로고침 후에 반영**됨(E-012). 업로드 후 반드시 새로고침해서 실제 등록 여부 확인.
- **URL로 cntId 바꿔도 콘텐츠 전환 안 됨**(스테일) → 스페이스 전환은 목록에서 카드 클릭 + 필요시 새로고침.
- **오래 작업하면 업로드가 실패**할 수 있음 → 재로그인.
- **과장 금지**: 재현 안 되는 결함(간헐 크래시 등)은 정직하게 표기해야 개발사 판정에서 살아남음. E-019처럼 원인 불명확하면 철회하고 요확인으로.
- **결함이 승부처**: 완성도·기법은 이미 확보. 재로그인 후 **보통↑ 결함(비율 검증·크래시 재현)** 몇 건 더 잡는 게 최우선.
