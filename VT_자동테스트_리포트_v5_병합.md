## 🔴 심각 (Critical)

### E-018 [수동] 배치 업로드에 미지원 포맷 1장만 섞여도 정상 파일까지 전량 유실 ★v5 신규·데이터 손실
- **현상**: 다중선택 배치 업로드에 **미지원 포맷(WebP/GIF = 서버가 빈 jobId로 거부) 파일이 1장이라도 포함**되면, 같은 배치의 **정상 파일까지 전부 등록 실패**(데이터 손실). 실패 사유 안내 없이 "업로드 실패" 토스트만 뜨고 빈 유령 카테고리(E-011)가 남음.
- **근본원인 = 통제 실험 8종으로 격리**(콘텐츠·용량·개수·순서 교란 전부 제거):
  | # | 배치 구성 | 거부(webp/gif) | 해상도실패 | 결과 |
  |---|---|:---:|:---:|---|
  | 1 | 정상 5장 / 정상 10장 | ✗ | ✗ | ✅ 전부 성공 (개수·동시성 무죄) |
  | 2 | 정상2 + 해상도실패6 (9장) | ✗ | O | ✅ 정상분 생존(부분 성공) → 해상도실패는 배치 안 깨뜨림 |
  | 3 | **정상2 + webp1 (거부단독)** | **O(1)** | ✗ | ❌ **전량 몰살** ← 거부 1장 단독으로도 |
  | 4 | 정상2 + 해상도1 + webp1 (최소재현) | O(1) | O(1) | ❌ 전량 몰살 |
  | 5 | 정상4 + 해상도4 + 거부2 (10장) | O(2) | O(4) | ❌ 전량 몰살 |
  | 6 | 정상5 + 해상도5 + 거부2 (12장 뒤섞기) | O(2) | O(5) | ❌ 전량 몰살 |
- **단일 법칙**: **배치에 거부 파일(빈 jobId)이 1장이라도 있으면 정상 파일까지 전부 실패.** 해상도실패만 있는 배치는 정상분 생존(#2). 파일 개수·용량·동시성·업로드 순서 전부 반증.
- **추정 메커니즘**: 거부는 `jobId=(빈값)` 반환 → 빈 jobId를 폴링·집계하는 배치 완료 핸들러가 깨지며 **이미 성공한 씬 등록까지 롤백**.
- **실사용 피해**: 사진 폴더를 통째로 드래그할 때 스크린샷 WebP 1장만 섞여도 멀쩡한 파노라마 전부 유실. **재현 100%**, 순서 무관.
- 증적: 통제 매트릭스, 콘솔 빈 jobId, "업로드 실패" 토스트, 빈 유령 카테고리(뒤죽박죽/크래시테스트), 개별 재업로드 교차확인. (xlsx ID **E-018**)
- ※ **A-06과 부분 보정**: 자동(A-06)은 "무효 파일 비동기 조용히 폐기(silent)"로 봤으나, **UI는 '업로드 실패' 토스트를 표시**함(정빈 실측). 다만 사유·부분성공 여부는 미고지.

### A-01 [자동] 비활성(미게시) 스페이스 콘텐츠가 비인증 뷰어 API로 유출 ★신규·직접 실측
- `active=false`(미게시) 스페이스에 씬 생성 후 **토큰 없이** `GET vtapi.visitinc.kr/vt?VT=153dfe4` → **HTTP 200으로 전체 콘텐츠 서빙**(스페이스 제목·씬 정보·**360 타일 저장경로**).
- **의미**: 게시 여부와 무관하게 **VT 키만 알면 누구나 미게시 스페이스 열람**. 키는 7자리 hex라 추측·열거 리스크도 있음.
- **주의**: 관리자 엔드포인트(`content/auth?cntId=`)는 토큰 없이 401(접근제어 정상)이지만, **뷰어 데이터 엔드포인트(`/vt?VT=`)는 별개로 미게시 콘텐츠 노출**.
- 증적: verify_v1 실측(사전 active=false 확인 → 씬 생성 → 비인증 200·콘텐츠·타일경로 노출 → 정리). 게시상태 게이팅 필요.
- **남은 과제(수동)**: 로그아웃 브라우저에서 `/vt?VT=<미게시 키>` 육안 재현 후 증적(고배점, 개발사 우선 보고).

## 🟠 보통 (Major)

### E-019 [수동] 특정 해상도 파노라마가 업로드 처리 실패 (콘텐츠 무관) ★v5 신규
- **현상**: 정상 규격(2:1·유효 JPEG·적정 용량) 파노라마인데도 **≈1만×5천(10771×5385 등) 해상도는 처리 실패**(jobId 발급 후 씬 미등록), 7180×3590·11968×5984는 통과.
- **근본원인 = 해상도 (A/B 스왑 통제 실험)**: 동일 콘텐츠에서 **해상도만 스왑하면 성패가 반전** → 해상도가 유일 원인. 용량(6MB통과/8MB실패/24MB통과)·비율(전부 2.00)·홀짝(짝수 10770×5384도 실패) 전부 반증.
- **피드백**: "업로드 실패" 토스트는 뜨나 **왜 실패인지·허용 해상도 안내 없음**, 사전검증도 없음. (정확한 임계 해상도는 개발사 확인 영역)
- 증적: A/B 스왑 결과표, 콘솔 jobId 발급 vs 씬 미등록. (xlsx ID **E-019**)

### E-009 [양쪽] 초소형(100×50) 이미지가 씬으로 통과 → 뷰어 검은 화면
- 최소 크기/비율 검증 부재. **[자동]** API에서 tiny가 실제 씬으로 생성됨 확인 + **[수동]** 뷰어에서 완전 검은 화면 육안 확인(D3 재확인). 안내·플레이스홀더 없음.

### E-010 [수동] 업로드 용량 상한이 안내와 5배+ 불일치
- 안내는 "5MB까지"인데 **실측 24MB까지 통과**(25/30MB.jpg=실24MB), 35MB 거부. 상한 미정의·안내 오류. (자동은 압축으로 파일이 작아 미검출 — 수동 고유 성과)

### E-012 [양쪽] 업로드 직후 에디터가 "이미지 없음"으로 잘못 표시, 새로고침 필요
- 비동기 job 반영 지연. **[수동]** 육안 확인 + **[자동]** 이 현상이 자동 렌더 캡처를 막았음(같은 결함을 양쪽이 독립 확인).

### A-02 [자동] 로그인 실패(미가입 이메일)에 HTTP 201 + "id not exist" → 성공코드화 + 사용자 열거
- 정상형식 미가입 이메일+임의 비번 → **201 + `{"message":"id not exist"}`**. ① 인증 실패를 2xx로 통보(401이어야) ② 가입/미가입 구분 노출 = 사용자 열거. 수정: 401 + 일반 메시지.

### A-03 [자동] 입력검증 미비로 잘못된 요청이 4xx 대신 HTTP 500
- 초장문 파일명 업로드 → `POST /scene/autoNorth` **500**(임계 50자 202/100자 500). PUT /content 불완전 body → **500**(생성 POST /content는 400 검증 존재 → 검증 비대칭).

### A-04 [자동] 보안 응답 헤더 6종 전무 + 인증 토큰 localStorage 평문 저장
- CSP·X-Frame-Options·HSTS·X-Content-Type-Options·Referrer-Policy·Permissions-Policy 전부 부재. `x-powered-by: Express` 노출. 토큰이 **localStorage 평문** → CSP 부재와 결합 시 XSS로 세션 탈취.

## ✅ PASS (커버리지 증빙 · 양쪽 공통)
- 권한/IDOR 차단(회사 GET: own 200 / 미소속 code103), 비인증 API 401.
- 로그인 입력 서버측 경계검증(400), POST /content 생성 검증(400).
- 나쁜 파일(0바이트·손상·텍스트위장·webp·bmp) 최종 거부(단일/개별 업로드 시).
- **정상 파일만의 배치(5·10장) 부분성공 정상 작동** — 배치 자체는 건전, E-018은 "거부 파일 혼입" 특정 조건.
- ⚠ 관리자 엔드포인트 접근제어는 정상이나 **뷰어 엔드포인트는 A-01로 유출** — "접근제어 PASS"는 관리 API에 한정.
